---
title: "Quickstart"
description: "Set up Ruby in your GitHub Actions workflow in under a minute"
---

<Note>
  Always use `ruby/setup-ruby@v1`. The `v1` branch automatically receives bug fixes, new Ruby versions, and new features. Pinning to a specific release or commit means you must update it manually whenever a new Ruby version is needed.
</Note>

<Steps>
  <Step title="Create your workflow file">
    Create `.github/workflows/test.yml` in your repository. If the directory does not exist yet, create it.

    ```bash
    mkdir -p .github/workflows
    touch .github/workflows/test.yml
    ```
  </Step>

  <Step title="Add the setup-ruby step">
    Add the following workflow to `.github/workflows/test.yml`. This example checks out your code, sets up Ruby 3.4 with automatic gem caching, and runs your test suite.

    ```yaml .github/workflows/test.yml
    name: Tests
    on: [push, pull_request]
    jobs:
      test:
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v6
          - uses: ruby/setup-ruby@v1
            with:
              ruby-version: '3.4'
              bundler-cache: true
          - run: bundle exec rake
    ```

    `bundler-cache: true` automatically runs `bundle install` and caches the installed gems. You do not need a separate `bundle install` step.
  </Step>

  <Step title="Run tests with bundle exec rake">
    The `bundle exec rake` command runs your default Rake task inside the Bundler environment. Replace this with your actual test command if needed, for example:

    ```yaml
    - run: bundle exec rspec
    ```

    or

    ```yaml
    - run: bundle exec rails test
    ```
  </Step>

  <Step title="(Optional) Use a .ruby-version file for automatic version detection">
    If you commit a `.ruby-version` file to your repository, you can omit the `ruby-version` input entirely. The action reads it automatically.

    ```bash
    echo "3.4" > .ruby-version
    git add .ruby-version
    git commit -m "Add .ruby-version"
    ```

    Your workflow step then becomes:

    ```yaml
    - uses: ruby/setup-ruby@v1
      with:
        bundler-cache: true
    ```

    The action also supports `.tool-versions` (used by asdf) and `mise.toml`. If none of these files exist and `ruby-version` is not set, the action will fail.
  </Step>
</Steps>

## Complete minimal workflow

Here is the full minimal workflow from step 2 for easy copying:

```yaml .github/workflows/test.yml
name: Tests
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
      - uses: ruby/setup-ruby@v1
        with:
          ruby-version: '3.4'
          bundler-cache: true
      - run: bundle exec rake
```
