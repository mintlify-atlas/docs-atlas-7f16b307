Windows CI can be challenging if you are not familiar with Windows development. It is recommended to get your build working on Ubuntu or macOS first before attempting Windows.

## Supported Windows runners

`setup-ruby` supports these Windows GitHub-hosted runner images:

- `windows-2022`
- `windows-2025`
- `windows-11-arm`

## Ruby variants on Windows

Windows offers three distinct Ruby builds, each using a different compiler toolchain:

| Variant | Toolchain | Description |
| ------- | --------- | ----------- |
| `mingw` | MSYS2/MinGW | Standard RubyInstaller2 build using GCC via MinGW |
| `ucrt`  | MSYS2/UCRT64 | RubyInstaller2 build targeting the Universal C Runtime; `head` also uses UCRT64 |
| `mswin` | MSVC/VS 2022 | Build using Microsoft Visual C++ (Visual Studio 2022) |

<Note>
Only release versions published by [RubyInstaller](https://rubyinstaller.org/downloads/archives/) are available on Windows. The `mingw`, `ucrt`, and `mswin` master builds are generated by [ruby-loco](https://github.com/MSP-Greg/ruby-loco).
</Note>

## Toolchain setup

### Ruby >= 2.4: MSYS2

For Ruby 2.4 and later, MSYS2 is downloaded, extracted, and prepended to `Path` — mirroring what RubyInstaller2 does when you install Ruby on a local Windows machine. The `ridk enable` environment is activated automatically so that native extension compilation works without additional steps.

### Ruby < 2.4: DevKit (MSYS1)

For Ruby versions older than 2.4, the legacy DevKit MSYS tools are installed and prepended to `Path`. The following environment variables are also set: `CC`, `CPP`, `CXX`, `MAKE`, `RI_DEVKIT`, and `SSL_CERT_FILE`.

<Warning>
Ruby 2.4 and earlier, along with the OpenSSL 1.0.2 version they require, are both end-of-life and considered insecure. Avoid using them for new projects.
</Warning>

## The `windows-toolchain` input

The `windows-toolchain` input controls how the toolchain is set up. It accepts two values:

| Value | Behavior |
| ----- | -------- |
| `default` | Installs MSYS2 (if not already present) and the appropriate build tools and packages for the selected Ruby variant. Sets environment variables via `ridk` or `vcvars64.bat`. |
| `none` | Only adds the Ruby `bin` directory to `Path`. No build tools are installed and no environment variables are changed. |

Use `windows-toolchain: none` when you do not need to compile native extensions and want to keep the environment minimal.

```yaml
- uses: ruby/setup-ruby@v1
  with:
    ruby-version: '3.4'
    windows-toolchain: none
```

## Default shell is PowerShell

Unlike Ubuntu and macOS runners, the default shell on Windows is PowerShell, not Bash. This can cause problems with multi-line `run` scripts that use Bash syntax.

To use Bash for a specific step, add `shell: bash`:

```yaml
- name: Run tests
  shell: bash
  run: bundle exec rake
```

Or set the default shell for the entire job:

```yaml
jobs:
  test:
    runs-on: windows-latest
    defaults:
      run:
        shell: bash
```

## Multiple compiler toolchains on PATH

The Windows runner image includes multiple compiler toolchains, which means `PATH` may contain entries from several different toolchains simultaneously. If a tool resolves to an unexpected location, use `where.exe` to debug which binary is being used:

```yaml
- name: Debug PATH
  run: |
    where.exe ruby
    where.exe gcc
    where.exe make
```

## Bundler on Windows

Use Bundler 2.2.18 or later on Windows. Older versions have known bugs that cause failures specific to Windows. To ensure a recent Bundler version is used:

- Do not set the `bundler:` input (let the action choose the version automatically).
- Make sure your checked-in `Gemfile.lock` does not contain a `BUNDLED WITH 1.x.y` section.

## Example workflow

```yaml
name: Tests
on: [push, pull_request]
jobs:
  test:
    runs-on: windows-2022
    steps:
      - uses: actions/checkout@v6
      - uses: ruby/setup-ruby@v1
        with:
          ruby-version: '3.4'
          bundler-cache: true
      - name: Run tests
        shell: bash
        run: bundle exec rake
```

### Matrix across variants

```yaml
name: Tests (Windows variants)
on: [push, pull_request]
jobs:
  test:
    strategy:
      fail-fast: false
      matrix:
        ruby: [mingw, ucrt, mswin]
    runs-on: windows-2022
    steps:
      - uses: actions/checkout@v6
      - uses: ruby/setup-ruby@v1
        with:
          ruby-version: ${{ matrix.ruby }}
          bundler-cache: true
      - name: Run tests
        shell: bash
        run: bundle exec rake
```

## Installing additional packages

When compiling native extensions on `windows-2022`, the packages needed to build Ruby itself are included. If you need additional MSYS2 packages, install them with [setup-ruby-pkgs](https://github.com/ruby/setup-ruby-pkgs) or via MSYS2's `pacman`.

<Tip>
If you are hitting failures when installing or updating Ruby standard library extension gems, those gems may require additional MSYS2 packages that are not bundled with the runner image.
</Tip>
