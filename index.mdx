Set up Ruby for your GitHub Actions workflows in seconds. `setup-ruby` downloads a prebuilt Ruby binary — MRI, JRuby, or TruffleRuby — and adds it to the `PATH` with no extra packages required.

```yaml
- uses: ruby/setup-ruby@v1
  with:
    ruby-version: '3.4'
    bundler-cache: true
```

## Why setup-ruby?

- **Fast** — prebuilt binaries are downloaded and extracted in ~5 seconds, no compilation required
- **Broad version support** — MRI from 1.9.3 through head, JRuby 9.1+, and TruffleRuby 19.3+
- **Automatic Bundler management** — installs the right Bundler version and caches gems automatically
- **Version file support** — reads `.ruby-version`, `.tool-versions`, or `mise.toml` automatically
- **Cross-platform** — works on all GitHub-hosted Ubuntu, macOS, and Windows runners

## Getting started

<Steps>
  <Step title="Add the action to your workflow">
    Use `ruby/setup-ruby@v1` in your workflow file. Always use the `v1` branch to get automatic updates.

    ```yaml .github/workflows/test.yml
    - uses: ruby/setup-ruby@v1
      with:
        ruby-version: '3.4'
    ```
  </Step>
  <Step title="Enable Bundler caching (recommended)">
    Add `bundler-cache: true` to automatically run `bundle install` and cache the result.

    ```yaml .github/workflows/test.yml
    - uses: ruby/setup-ruby@v1
      with:
        ruby-version: '3.4'
        bundler-cache: true
    ```
  </Step>
  <Step title="Run your tests">
    Execute your test suite using `bundle exec`.

    ```yaml .github/workflows/test.yml
    - run: bundle exec rake
    ```
  </Step>
</Steps>

<Tip>
  Prefer `ruby/setup-ruby@v1` over pinning to a specific version or commit. The `v1` branch automatically receives bug fixes, new Ruby versions, and new features.
</Tip>
