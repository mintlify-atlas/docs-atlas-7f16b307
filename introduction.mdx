`setup-ruby` is a GitHub Action that downloads a prebuilt Ruby binary and adds it to the `PATH`. It supports MRI, JRuby, and TruffleRuby and takes about 5 seconds to run — no compilation, no extra packages.

```yaml
- uses: ruby/setup-ruby@v1
  with:
    ruby-version: '3.4'
    bundler-cache: true
```

## Key benefits

- **Fast** — prebuilt binaries download, extract, and are ready in ~5 seconds
- **Zero extra packages** — no system dependencies to install before use
- **Broad implementation support** — MRI, JRuby, TruffleRuby, and TruffleRuby+GraalVM
- **Automatic Bundler management** — installs the correct Bundler version based on your `Gemfile.lock`, and caches gems with `bundler-cache: true`
- **RubyGems control** — optionally update RubyGems to `latest` or a specific version
- **Version file support** — reads `.ruby-version`, `.tool-versions`, or `mise.toml` automatically when `ruby-version` is not set

## How it works

When the action runs, it:

1. Resolves the requested Ruby version from the input or from a version file in your repository
2. Downloads the prebuilt binary from the appropriate release source
3. Extracts it into the GitHub Actions tool cache
4. Adds the Ruby binary directory to the `PATH`
5. Installs or configures Bundler based on your `Gemfile.lock`
6. Optionally runs `bundle install` and caches the result

Prebuilt releases for Ubuntu and macOS come from [ruby-builder](https://github.com/ruby/ruby-builder). Windows releases come from [RubyInstaller2](https://github.com/oneclick/rubyinstaller2). The `mingw`, `ucrt`, and `mswin` head builds are generated by [ruby-loco](https://github.com/MSP-Greg/ruby-loco). Development head builds are generated by separate builder repositories for each implementation.

The tool cache means subsequent runs on the same runner image skip the download entirely if the version is already cached.

## Versioning: `@v1` vs pinning

<Note>
  Always use `ruby/setup-ruby@v1`. Pinning to a specific release or commit SHA means you will not automatically receive new Ruby versions, bug fixes, or platform support updates — and you will need to update the pin manually over time.
</Note>

`ruby/setup-ruby@v1` follows a moving branch that tracks the latest stable release. This follows the [GitHub Actions versioning recommendations](https://github.com/actions/toolkit/blob/master/docs/action-versioning.md). If you pin to `v1.2.3` or a commit SHA, only the Ruby versions available at that point in time will work.

## Supported Ruby implementations

| Implementation | Description |
| --- | --- |
| `ruby` (MRI) | The reference Ruby interpreter. Versions from 1.9.3 through 4.0.2, plus `head` and special builds |
| `jruby` | Ruby on the JVM. Versions from 9.1.17.0 through 10.0.4.0, plus `head` |
| `truffleruby` | Ruby on GraalVM Community. Versions from 19.3.0 through 33.0.1, plus `head` |
| `truffleruby+graalvm` | Ruby on GraalVM Enterprise/Oracle. Versions from 21.2.0 through 33.0.1, plus `head` |
